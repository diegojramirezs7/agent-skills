# Process skills
These skills are meant to help users refine ideas, clarify requirements, create specifications, do the initial system design, and write a detail implementation plan.
All of these skills have in common that are related to designing and creating implementation documents for a codebase, but none of them involve coding. It just helps clarify the process and implementation details.


How to use process skills
Process skills are: 
- refine-project (rough notes to concrete idea)
- system-design (from concrete project idea to components, data model, api, frontend structure, etc)
- refine-feature (from rough idea notes to concrete feature idea with plan -- tech agnostic)
- feature-implementation-plan (from concrete feature idea to codebase-specific plan and spec)
